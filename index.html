Frequently Asked Questions

It is a long established fact that a reader will be distracted by the readable content of a page when looking at its layout.

CSS Box Model : All the element present in html follows css box module. The CSS box model is essentially a box that binds every HTML element in it. The css box model consists of margins, borders, padding, and the actual content. When we set the width and height properties of an element with CSS, we just set the width and height of the content area. To calculate the full size of an element, we must also add padding, borders and margins.

The total width of an element should be calculated like this: Total element width = width + left padding + right padding + left border + right border + left margin + right margin The total height of an element should be calculated like this: Total element height = height + top padding + bottom padding + top border + bottom border + top margin + bottom margin.

Semantic Tags: What They Are and How to Use Them! Semantic elements have meaningful names which tells about type of content. For example header, footer, table, … etc . A semantic element clearly describes its meaning to both the browser and the developer.These tags tell the browser how to display the content on the page.That explicit information helps robots/crawlers like Google and Bing to better understand which content is important, which is a subsidiary, which is for navigation, and so on.

Grid and flexbox : CSS Grid and Flexbox are layout models that share similarities and can be used together. The key difference is that CSS Grid can be used to create two-dimensional layouts, while Flexbox can only be used to create one-dimensional layouts. That means you can place components along the X- and Y-axis in CSS Grid and only one axis in Flexbox. Making it easier to design web pages without having to use floats and positioning. Like tables, grid layout allow us to align elements into columns and rows.

Bootstrap vs Tailwind : To create an interface between computers and people, developers use front-end frameworks. These frameworks are designed and developed to create sites for people. They allow developers and users to have adequate control. Bootstrap and Tailwind are also two such CSS front-end frameworks used on a large scale.Bootstrap is what I'd call a Component-based Framework. It's designed to help developers quickly bootstrap a typical website or app and so comes with a set of pre-made components, along with some utilities for spacing etc. Other similar frameworks include Bulma and Foundation. TailwindCSS is a Utility-first Framework. In contrast to Bootstrap, which provides a few utilities, Tailwind is all utilities. The core framework has no pre-made components, so you'll need to build your own.
